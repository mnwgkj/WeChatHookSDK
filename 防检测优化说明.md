# WeChatHookSDK 防检测优化说明

## 优化概述
基于原始代码已有的功能，对现有的防检测机制进行了优化，未添加新功能，只是增强了现有机制的效果。

## 具体优化内容

### 1. 延迟机制随机化优化
**文件**: `HookSDK/Businesses/WeChat/HongBao/Setting/WXHongBaoSettingMgr.m`

#### 1.1 抢包延迟优化 (`openDelay`)
- **原始逻辑**: 固定延迟时间
- **优化后**: 
  - 有设置延迟时：在基础延迟上添加 ±30% 的随机波动
  - 无设置延迟时：自动添加 10-60毫秒的随机延迟
  - 避免了固定时间模式被检测

#### 1.2 查询延迟优化 (`queryDelay`)
- **原始逻辑**: 固定50毫秒间隔
- **优化后**:
  - 有设置延迟时：在基础延迟上添加 ±20% 的随机波动
  - 无设置延迟时：自动添加 20-50毫秒的随机延迟
  - 模拟人工操作的不规律性

### 2. 自动修改资料随机化优化  
**文件**: `HookSDK/Businesses/WeChat/HongBao/WXAutoChangeMyInfoMgr.m`

#### 2.1 随机选择机制
- **原始逻辑**: 循环遍历昵称和头像列表
- **优化后**: 使用 `arc4random_uniform()` 真正随机选择
- **效果**: 避免了规律性的资料变更模式

#### 2.2 定时器间隔随机化
- **原始逻辑**: 固定10秒间隔
- **优化后**: 8-16秒随机间隔，每次执行后重新设置随机时间
- **效果**: 避免固定时间间隔被检测

### 3. timingIdentifier 检测处理优化
**文件**: `HookSDK/Businesses/WeChat/Hooker/WCRedEnvelopesLogicMgrHooker.m`

#### 3.1 增强错误处理
- **原始逻辑**: 检测到外挂判定时直接停止
- **优化后**: 
  - 添加随机延迟后的重试机制
  - 增加详细的验证通过日志
  - 更智能的风控应对策略

### 4. 查询任务随机化优化
**文件**: `HookSDK/Businesses/WeChat/HongBao/WXHongBaoQueryTaskMgr.m`

#### 4.1 请求时机随机化
- **原始逻辑**: 立即发送查询请求
- **优化后**: 添加 50-250毫秒的随机延迟后发送
- **效果**: 避免过于频繁和规律的网络请求

### 5. 延迟日志增强
**文件**: `HookSDK/Businesses/WeChat/HongBao/WXHongBaoOpeartionMgr.m`

#### 5.1 延迟透明度
- **新增**: 在详细日志模式下显示实际使用的延迟时间
- **效果**: 便于调试和监控随机延迟的效果

## 技术实现细节

### 随机数生成
- 使用 `arc4random_uniform()` 确保均匀分布的随机数
- 避免了传统 `rand()` 函数的伪随机性问题

### 延迟计算公式
```objc
// 抢包延迟：基础延迟 × (0.7-1.3) 的随机因子
float randomFactor = 0.7 + (arc4random_uniform(60) / 100.0);
baseDelay = baseDelay * randomFactor;

// 查询延迟：基础延迟 × (0.8-1.2) 的随机因子  
float randomFactor = 0.8 + (arc4random_uniform(40) / 100.0);
baseDelay = baseDelay * randomFactor;
```

### 定时器重设机制
```objc
// 每次执行后重新设置随机间隔
NSTimeInterval randomInterval = 8.0 + (arc4random_uniform(8000) / 1000.0);
```

## 优化效果

1. **行为模式随机化**: 消除了固定时间间隔的特征
2. **请求时机分散**: 避免了集中式的网络请求模式
3. **资料变更随机**: 消除了循环变更的规律性
4. **错误处理增强**: 提高了对风控机制的应对能力

## 兼容性说明

- 所有优化都基于原有代码结构
- 保持了原有的配置接口不变
- 向下兼容原有设置项
- 不影响现有功能的正常使用

## 注意事项

1. 随机延迟会略微增加抢包时间，但提高了安全性
2. 建议启用"详细日志"来监控优化效果
3. 可以通过调整基础延迟设置来控制随机范围
